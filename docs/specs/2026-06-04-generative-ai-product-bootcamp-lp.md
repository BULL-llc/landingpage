# 生成 AI プロダクト開発ブートキャンプ LP 仕様書

## 概要

生成 AI とクラウドを使い、2 日間でアイデアを動くプロダクトにする力を身につける人材育成プログラムのランディングページを作成する。
主目的は参加申し込みであり、高単価 B2B 研修として信頼感と実践感を両立する。

## ユーザーストーリー

- 企業の DX、人材育成、事業開発担当者として、生成 AI 時代のプロトタイピング人材を育てる研修内容を理解し、参加申し込みを行いたい。
- ソフトウェアエンジニアとして、AI とクラウドを活用した実践的な開発演習の到達点とカリキュラムを確認したい。
- 非エンジニアとして、PowerPoint レベルのアイデアを動くプロトタイプへ落とし込めるようになる学習体験を知りたい。

## 受け入れ基準

- [ ] ファーストビューで「生成 AI プロダクト開発ブートキャンプ」と「2 日間で、アイデアを動くプロダクトへ」が伝わる。
- [ ] 主 CTA は「参加申し込み」とし、クリックで申し込みフォームまでスクロールする。
- [ ] 料金は「2 日間 / 1 名 200,000 円（税別）」として表示する。
- [ ] 課題、受講後の姿、カリキュラム、学習ジャーニー、2 日間の流れ、料金、FAQ、申し込みフローを含める。
- [ ] FAQ は開閉できる。
- [ ] 申し込みフローは 1 問 1 答形式で、担当者名、会社名、メール、人数、参加者構成、社内課題、希望時期、同意チェック、送信完了状態が動作する。
- [ ] 人数は通常枠を 6 名までとし、7 名以上は応相談として表示する。
- [ ] 社内トレーナー育成、キャッシュバック、社内展開サポートの訴求は表示しない。
- [ ] デスクトップとモバイルで読める余白、文字サイズ、CTA 配置を保つ。

## 非機能要件

- パフォーマンス: 主要画像はローカル配信し、外部 CDN への依存を避ける。
- セキュリティ: 申し込みフォームはデモ送信のみとし、個人情報を永続化しない。
- アクセシビリティ: フォームラベル、フォーカス表示、FAQ の `aria-expanded` を設定する。

## 技術設計

- データモデル: 永続化なし。フォーム入力はブラウザ内の一時状態のみ。
- API エンドポイント: なし。
- UI コンポーネント: ヘッダー、ヒーロー、課題セクション、成果セクション、カリキュラム、学習ジャーニー、タイムライン、料金、FAQ、申し込みフロー。
- 実行方式: `packages/frontend` に依存追加なしの静的フロントエンドを配置し、Bun の軽量サーバーでローカル確認する。

## スコープ外

- 本番の申し込み API 連携。
- 決済処理。
- 管理画面。
- React / Vite への移行。
- 社内トレーナー育成やキャッシュバックの訴求。
